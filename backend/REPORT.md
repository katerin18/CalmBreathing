# Краткий отчет по проекту Calm Breathing Backend

Дата: 14.05.2026

## Цель этапа
Реализовать backend API для мобильного приложения дыхательных упражнений и подготовить документацию для интеграции фронтенда с бэкендом.

## Выполненные работы
1. Построена архитектура backend на Spring Boot с разделением на слои: controller, service, repository, security, error handling.
2. Реализована JWT-аутентификация с access/refresh токенами.
3. Реализованы основные пользовательские сценарии:
   - регистрация,
   - вход,
   - обновление access token,
   - выход,
   - получение и обновление профиля,
   - смена пароля.
4. Добавлен модуль замеров пульса:
   - сохранение замера,
   - получение последнего замера пользователя.
5. Настроены миграции БД через Liquibase и структура таблиц для пользователей, ролей, токенов и замеров.
6. Подготовлена и актуализирована документация по API и архитектуре.
7. В основной README добавлен отдельный раздел по передаче данных с фронта на бэкенд:
   - форматы запросов (JSON и query/form params),
   - обязательные заголовки,
   - примеры payload,
   - поток refresh токена,
   - единый формат ошибок.

## Текущий статус
- Backend готов к интеграции с мобильным/веб-клиентом.
- Контракт обмена данными между фронтом и бэком формализован.
- Проект готов к локальному запуску, тестированию и дальнейшему расширению.

## Риски и зоны развития
1. Добавить rate limiting для auth endpoint.
2. Реализовать email verification и password reset flow.
3. Расширить интеграционные и security-тесты.
